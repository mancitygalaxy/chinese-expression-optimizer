# 业务文档改写示例

## 示例一：减少翻译腔

原文：

```markdown
我们需要基于现有信息来进行下一步策略制定。
```

改写：

```markdown
我们要根据现有信息判断下一步怎么做。
```

原因：中文里直接说动作更自然，“基于……进行……”会让句子显得绕。

## 示例二：先讲动作和结果

原文：

```markdown
该能力通过任务流转机制实现业务闭环。
```

改写：

```markdown
系统会把待处理事项分配给对应负责人，并跟踪处理结果，确保每个问题都有明确结论。
```

原因：“业务闭环”太泛，读者看不出具体发生什么。改写后能看到动作、负责人和结果。

## 示例三：少用不必要英文

原文：

```markdown
我会先同步 current status，再给 recommendation。
```

改写：

```markdown
我会先同步当前情况，再给建议。
```

原因：这里的英文没有增加精度，反而打断阅读。
