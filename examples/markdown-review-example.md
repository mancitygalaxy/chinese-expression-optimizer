# Markdown 审校输出示例

## 待审校片段

```markdown
本方案不是恢复旧链路，而是新增批量读取能力。通过 Agent 对 memory modules 进行读取，实现用户信息的结构化注入。
```

## 审校建议

**总体判断**

这段话能看出方案方向，但否定式开头和中英混用较多，业务读者需要多读几遍才能理解。建议先正向说明本方案做什么，再把旧链路作为边界补充。

**高优先级问题**

- “不是……而是……”把边界放在了主句位置，正文重点不够直接。
- `memory modules` 没有中文解释，混合读者不一定知道它指什么。
- “结构化注入”偏工程黑话，可以改成更具体的动作。

**可替换片段**

```markdown
本方案采用批量读取能力，Agent 按需读取指定的记忆模块 `memory modules`，并把用户信息整理成模型可直接使用的上下文。旧链路只作为迁移边界说明，不进入首期主流程。
```

**仍需确认**

- `memory modules` 是否是正式字段名。
- “旧链路”的具体范围是否已有统一口径。
